# 클라우드 셋업 가이드 (전부 무료)

이 프로젝트는 **무료 범위 안에서 전부 동작**한다. 지금 당장 필요한 건 **Firebase 하나뿐**이다.

| 서비스 | 용도 | 무료 | 카드 |
|---|---|---|---|
| Firebase **Spark** | RTDB + 익명 인증 | 동시접속 100, 1GB, 10GB/월 | 불필요 |
| Cloudflare **Pages** | 호스팅(배포) | 무제한 정적 | 불필요 |
| Cloudflare **Workers** | 이미지 업로드 중계 | 10만/일 | 불필요 |
| Cloudflare **R2** | 이미지 저장(선택) | 10GB, egress 무료 | ⚠️ 카드 등록 요구($0 청구) |

> 작품 이미지는 당장은 **URL 붙여넣기**로 처리 → R2 없이 0원. R2는 정말 필요할 때만.

---

## 1. Firebase (지금 필요 — 약 3분)

1. https://console.firebase.google.com 접속 (babosam@gmail.com 로그인)
2. **프로젝트 만들기** → 이름 입력(예: `art-auction`) → Google 애널리틱스는 꺼도 됨 → 생성
   - 요금제는 기본 **Spark(무료)**. Blaze로 업그레이드하지 말 것.
3. 왼쪽 **빌드 → Realtime Database** → **데이터베이스 만들기**
   - 위치: 아무거나(예: `asia-southeast1` 또는 미국)
   - 보안 규칙: **잠금 모드**로 시작(나중에 앱 규칙으로 교체)
4. 왼쪽 **빌드 → Authentication** → **시작하기** → **Anonymous(익명)** 사용 설정 → 저장
5. 왼쪽 위 **프로젝트 개요 옆 톱니 → 프로젝트 설정** → 아래로 스크롤 → **내 앱**에서 **웹(`</>`)** 추가
   - 앱 닉네임 입력 → 등록 → 표시되는 `firebaseConfig` 값 복사

복사할 값 (`.env`에 입력):

```
VITE_FIREBASE_API_KEY=        # apiKey
VITE_FIREBASE_AUTH_DOMAIN=    # authDomain
VITE_FIREBASE_DATABASE_URL=   # databaseURL  ← Realtime Database 주소(중요)
VITE_FIREBASE_PROJECT_ID=     # projectId
VITE_FIREBASE_APP_ID=         # appId
VITE_USE_EMULATOR=false
```

> `databaseURL`이 config에 안 보이면 Realtime Database 페이지 상단의 `https://<...>.firebasedatabase.app` 주소를 쓴다.

검증:

```bash
node scripts/verify-slice.mjs   # ✅ Step 3 수직 슬라이스 통과 가 나오면 성공
```

## 2. Cloudflare Pages (배포 시 — 나중)

1. https://dash.cloudflare.com → Workers & Pages → **Create → Pages → Connect to Git**
2. 이 GitHub 저장소 선택 → 빌드 설정: 빌드 명령 `npm run build`, 출력 디렉터리 `dist`
3. 환경 변수에 `VITE_FIREBASE_*` 입력 → 배포

## 3. 이미지 업로드 (선택 — 나중)

- 기본: 교사가 이미지 **URL을 붙여넣기**(인프라 0).
- 강화: Cloudflare R2 + Worker 중계(`worker/`). R2는 카드 등록이 필요하나 무료 범위 내 $0.
